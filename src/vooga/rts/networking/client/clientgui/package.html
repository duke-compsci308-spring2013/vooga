This package is part of the CompSci 308 VOOGA Game Engine Project.

Class hierarchy

The classes from the view are commanded by ClientModel, which chooses the appropriate view to display to the user

Workflow

This package contains several GUI components that help put up A user friendly interface.
There is a view that corresponds to the server browser, in which the user chooses among different games to join. There is also a view that acts as a waiting lobby while the game does not start. Both of these views contain a table object that extend AbstractTableModel to display information.
The model of the client makes use of a ClientViewAdapter to handle the interactions between the view and the model

Opportunities for extension

ViewContainerPanel can be adapted to receive one interface and set the main panel, right, and left buttons accordingly. Other GUIs can be created and set through this class while the class would only need an interface reference from it
Also, the view adapter superclass can be used to meddle the actions between view and model.
