Class hierarchy

The classes from the view are commanded by ClientModel, which chooses the appropriate view to display to the user

Workflow

This package contains several GUI components that help put up A user friendly interface.
ImageHelper contains some static and overloaded methods that shortens the process of creating images.
There is a view that corresponds to the server browser, in which the user chooses among different games to join. There is also a view that acts as a waiting lobby while the game does not start. Both of these views contain a table object that extend AbstractTableModel to display information.

Opportunities for extension

ViewContainerPanel can be adapted to receive one interface and set the main panel, right, and left buttons accordingly. Other GUIs can be created and set through this class while the class would only neeed an interface reference from it
