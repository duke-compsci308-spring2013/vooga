This package is part of the CompSci 308 VOOGA Game Engine Project.

This package represents messages from the client to the server.

Class hierarchy

Each message in this package subclasses ClientInfoMessage. All of them use the method AffectServer (ConnectionThread thread, IThreadContainer server) which acts in some way on the thread and server.

Workflow

The ClientInfoMessage is created by the client and sent to the server. The IThreadContainer that receives the message then calls AffectServer() on the message, passing in itself and the thread that passed in the message.

Opportunities for extension

Current examples include Joining the lobby, removing the connection and logging out, or requesting a list of all the current lobbies. Extending ClientInfoMessage is as easy as writing a new message that uses AffectMessage.
